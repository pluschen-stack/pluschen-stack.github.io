栈和队列

栈

​ 栈是一种数据结构，遵循着LIFO(即后进先出)的方式。

​ 栈有很多方面的应用，比如：

​ 递归，一层层的调用并压入栈中，在调回时，一个个弹出栈。

​ 进制转换，比如十进制转换成二进制。转换的过程中是分别对二求余，显然最先修得的余数是个位，以此类推，这恰好可以使用栈来存储，一层层的压入栈中，在合成新的数时，从最上层取得0或1。

​ 处理表达式，对于形如(2 + ( (3 + 4 ) *(4 *5) ) )这样的中缀表达式就可以通过栈来计算结果，计算过程是遇到左括号则忽略，遇到操作对象或者操作符则压入栈中，遇到右括号时开始计算。

​ 这里解释一下均摊分析怎么执行的，就是将每个时间点的操作数平均到所有时间点上。

队列

​ 队列是一种数据结构，遵循着FIFO(即先进先出)的方式。

​ 队列的实际应用有很多，比如：

​ 假设你运营一个网站，你向成千上万的客户提供文件。但是由于你服务器性能所限，你并不能够一次处理掉所有的请求，你每次只能处理100次请求。那么一个有效的策略就是根据接受请求的顺序来处理请求，这样就需要遵循先进先出方式的队列。

​ 同样地，在一个多任务操作系统中，CPU不能够一次运行所用任务，所以任务必须要分批处理，并根据一些策略来执行。这种情况下，队列就是一种合适的选择。

​ 事实上，java的util库本身提供了Stack类和Queue接口，但是这并不意味着我们不需要了解这些是如何实现的，当我们了解了这些特定数据结构的实现后能够让我们对他们的效率有更好的理解。比如，用链表实现的数据结构增加很容易，但是随机查找需要很长的时间，数组实现的数据结构随机查找很快，但是增加需要更长的时间，并且会面临着修改数组长度的问题。这些都是不容忽视的部分，需要深入了解。
